Программа для работы с графами: поиск путей, минимальное остовное дерево и решение задачи коммивояжёра с использованием текстового интерфейса.

В проекте реализованы основные алгоритмы на графах: DFS и BFS обходы, алгоритмы Дейкстры и Флойда-Уоршелла для поиска кратчайших путей, алгоритм Прима для построения минимального остовного дерева, а также муравьиный алгоритм для приближенного решения задачи коммивояжёра.

## Функциональность

### Загрузка и управление графом

| Операция | Описание | Исключительные ситуации |
|----------|----------|-------------------------|
| `load_graph_from_file(g, filename)` | Загружает граф из файла с матрицей смежности. | Файл не существует, неверный формат, ошибка выделения памяти |
| `allocate_graph(g, size)` | Выделяет память под граф заданного размера. | Некорректный размер, недостаточно памяти |
| `export_graph_to_dot(g, filename)` | Экспортирует граф в формат DOT для визуализации. | Некорректный путь, ошибка записи |
| `free_graph(g)` | Освобождает память, занятую графом. | Передан NULL-указатель |

### Алгоритмы обхода

| Операция | Описание | Исключительные ситуации |
|----------|----------|-------------------------|
| `depth_first_search(g, start_vertex)` | Обход графа в глубину (DFS) с заданной вершины. | Некорректный номер вершины, граф NULL, ошибка памяти |
| `breadth_first_search(g, start_vertex)` | Обход графа в ширину (BFS) с заданной вершины. | Некорректный номер вершины, граф NULL, ошибка памяти |

### Поиск кратчайших путей

| Операция | Описание | Исключительные ситуации |
|----------|----------|-------------------------|
| `get_shortest_path_between_vertices(g, v1, v2)` | Находит кратчайший путь между двумя вершинами (алгоритм Дейкстры). | Некорректные номера вершин, граф NULL, путь не существует |
| `get_shortest_paths_between_all_vertices(g)` | Находит кратчайшие пути между всеми парами вершин (алгоритм Флойда-Уоршелла). | Граф NULL, ошибка выделения памяти |

### Минимальное остовное дерево

| Операция | Описание | Исключительные ситуации |
|----------|----------|-------------------------|
| `get_least_spanning_tree(g)` | Строит минимальное остовное дерево (алгоритм Прима). | Граф NULL, ошибка выделения памяти, граф несвязный |

### Задача коммивояжёра

| Операция | Описание | Исключительные ситуации |
|----------|----------|-------------------------|
| `solve_traveling_salesman_problem(g)` | Находит приближенное решение задачи коммивояжёра (муравьиный алгоритм). | Граф NULL, граф несвязный, отсутствие гамильтонова цикла |

### Структуры данных

| Структура | Описание |
|-----------|----------|
| `graph` | Основная структура графа: размер и матрица смежности |
| `stack` | Стек для DFS: вершины и размер |
| `queue` | Очередь для BFS: начало, конец и размер |
| `tsm_result` | Результат решения TSM: массив вершин и длина пути |

### Вспомогательные структуры

| Структура | Описание |
|-----------|----------|
| `stack_node` | Узел стека с данными и указателем на следующий узел |
| `queue_node` | Узел очереди с данными и указателем на следующий узел |

## Зависимости

* **Check** — для модульного тестирования
* **gcovr** — для генерации отчета о покрытии кода (опционально)

## Сборка

```bash
make all
```

Для сборки отдельных компонентов:

```bash
make s21_graph          # Только библиотека графа
make s21_graph_algorithms # Только библиотека алгоритмов
make s21_navigator      # Сборка исполняемого файла
```

Исполняемый файл будет создан в корневой директории под именем `s21_navigator`.

## Запуск

```bash
./s21_navigator
```

При запуске программа запросит имя файла с графом. Примеры файлов находятся в директории `examples/`.

## Формат входного файла

```
размер
матрица_смежности[0][0] матрица_смежности[0][1] ... матрица_смежности[0][n-1]
...
матрица_смежности[n-1][0] матрица_смежности[n-1][1] ... матрица_смежности[n-1][n-1]
```

Где `0` означает отсутствие ребра, положительное число — вес ребра. Граф должен быть неориентированным (матрица симметрична).

**Пример файла (3 вершины):**
```
3
0 5 3
5 0 4
3 4 0
```

## Вывод программы

При успешной загрузке графа программа последовательно выполняет:

1. **DFS обход** — для каждой вершины
2. **BFS обход** — для каждой вершины
3. **Алгоритм Дейкстры** — кратчайший путь между двумя случайными вершинами
4. **Алгоритм Флойда-Уоршелла** — кратчайшие пути между всеми парами вершин
5. **Алгоритм Прима** — минимальное остовное дерево
6. **Муравьиный алгоритм** — решение задачи коммивояжёра

## Параметры муравьиного алгоритма

Настройки алгоритма находятся в файле `s21_graph_algorithms.h`:

| Параметр | Значение | Описание |
|----------|----------|----------|
| `ANT_COUNT` | 50 | Количество муравьев в колонии |
| `TSM_ITERATIONS` | 100 | Количество итераций алгоритма |
| `ALPHA` | 1.0 | Влияние феромона на выбор пути |
| `BETA` | 2.0 | Влияние расстояния на выбор пути |
| `RATE` | 0.5 | Скорость испарения феромона |
| `Q` | 100.0 | Количество феромона, оставляемого муравьем |
